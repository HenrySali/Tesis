Resumen

El presente trabajo describe el diseño, la implementación y la evaluación de SmartTemp, un sistema basado en Internet de las Cosas (IoT) para el monitoreo continuo de la cadena de frío en un laboratorio de biología molecular. El sistema fue desarrollado con el objetivo de reemplazar el registro manual de temperatura —discontinuo, propenso a errores y sin capacidad de alerta— por una solución automatizada que garantice la trazabilidad exigida por la normativa vigente (ISO 15189:2012, Disposición ANMAT 2819/2004, Buenas Prácticas de Laboratorio).

La arquitectura del sistema sigue el modelo IoT de tres capas. La capa de percepción está compuesta por nodos sensores basados en microcontroladores ESP8266 con sensores de temperatura DS18B20 (±0,5 °C), que transmiten datos mediante WiFi utilizando los protocolos HTTP y MQTT. El firmware implementa un sistema de gestión offline con almacenamiento en LittleFS y un mecanismo jerárquico de 5 niveles de respaldo de timestamp que garantiza la continuidad del registro durante cortes de conectividad. La capa de procesamiento comprende un servidor Node.js con Express, una base de datos MySQL, un broker MQTT Mosquitto y un servidor WebSocket para distribución de datos en tiempo real. La capa de aplicación ofrece una interfaz web con dashboards de visualización, sistema de alertas por umbrales configurables con notificaciones automáticas vía WhatsApp y email, calibración por software, diagnóstico de red y gestión de equipos.

La evaluación se realizó durante 113 días (12 de marzo al 3 de julio de 2026) con 2 sensores y 43.430 mediciones registradas. La precisión se validó mediante calibración contra un patrón certificado Madgetech OctTemp (S/N 93230), obteniendo un error residual post-corrección de ±0,35 °C, cumpliendo el criterio de aceptación (≤ ±0,5 °C). La validación extendida con dataloggers UTRID-16 durante 25 días confirmó la concordancia mediante análisis de Bland-Altman (ICC > 0,75). El sistema demostró disponibilidad superior al 99% en modo de operación estable, recuperación del 100% de datos offline tras 1.539 reconexiones sin pérdida, y capacidad de detección de anomalías térmicas en tiempo real. Se registraron 155 notificaciones automáticas por WhatsApp ante eventos de desconexión y excursiones de temperatura.

El costo por punto de monitoreo (~7 USD) es significativamente inferior al de soluciones comerciales equivalentes (100-500 USD), con funcionalidades adicionales no disponibles en productos estándar como gestión offline de 5 niveles, configuración remota MQTT bidireccional y diagnóstico de red integrado.

Palabras clave: Internet de las Cosas, monitoreo de temperatura, cadena de frío, ESP8266, DS18B20, MQTT, WebSocket, laboratorio de biología molecular, sensores inalámbricos, Node.js, ISO 15189.

Abstract

This work describes the design, implementation, and evaluation of SmartTemp, an Internet of Things (IoT) system for continuous cold chain monitoring in a molecular biology laboratory. The system was developed to replace manual temperature logging —discontinuous, error-prone, and lacking alert capabilities— with an automated solution ensuring the traceability required by current regulations (ISO 15189:2022, ANMAT Disposition 2819/2004, Good Laboratory Practice).

The system architecture follows the three-layer IoT model. The perception layer consists of sensor nodes based on ESP8266 microcontrollers with DS18B20 temperature sensors (±0.5 °C), transmitting data via WiFi using HTTP and MQTT protocols. The firmware implements an offline management system with LittleFS storage and a hierarchical 5-level timestamp backup mechanism ensuring recording continuity during connectivity outages. The processing layer comprises a Node.js server with Express, a MySQL database, a Mosquitto MQTT broker, and a WebSocket server for real-time data distribution. The application layer provides a web interface with visualization dashboards, configurable threshold alerts with automatic WhatsApp and email notifications, software-based calibration, network diagnostics, and equipment management.

Evaluation was conducted over 113 days (March 12 to July 3, 2026) with 2 sensors and 43,430 recorded measurements. Precision was validated through calibration against a certified Madgetech OctTemp standard (S/N 93230), achieving a post-correction residual error of ±0.35 °C, meeting the acceptance criterion (≤ ±0.5 °C). Extended validation with UTRID-16 dataloggers over 25 days confirmed agreement through Bland-Altman analysis (ICC > 0.75). The system demonstrated availability exceeding 99% in stable operation mode, 100% offline data recovery after 1,539 reconnections with zero data loss, and real-time thermal anomaly detection capability. A total of 155 automatic WhatsApp notifications were registered for disconnection events and temperature excursions.

The cost per monitoring point (~7 USD) is significantly lower than equivalent commercial solutions (100-500 USD), with additional features not available in standard products such as 5-level offline management, bidirectional MQTT remote configuration, and integrated network diagnostics.

Keywords: Internet of Things, temperature monitoring, cold chain, ESP8266, DS18B20, MQTT, WebSocket, molecular biology laboratory, wireless sensors, Node.js, ISO 15189.
